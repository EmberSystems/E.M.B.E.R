File Usage:
core/
    command/
        parser.py > Takes Raw Input, and splits it into: { "command": "command-name", "args": ["", "test.lua"] }
        router.py > Recives parsed command (from parser.py) and decides what handler to call, ex:
                                                                                            send в†’ payload.py
                                                                                            start logserver в†’ logging.py
        commands/
                session.py > Handles: Kill Session, Reconnect
                             talks to session_manager
                payload.py > Handles: send payload, and resend last payload. Talks to payload_manager
                logging.py > Handles: start logserver, stop logserver, clear logs. Talks to log_manager
                system.py > Handles: refetch target info, debug mode. Talks to many managers < Handles global control commands that manage system-wide state, diagnostics, and environment                              operations across multiple components.
    managers/
        session_manager.py < Tracks current Session ID, Selected Payload, active device, last sent payload, and selected exploit
        payload_manager.py < Loads payload files, Makes sure they are compatable with the Selected Exploit, and sends the payload via device_manager
                             ^^^^^^^^^^^^^^^^^^^             ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                             payloads/Selected Exploit       Via a manifest file in payloads/metadata/manifest.json
        exploit_manager.py < Loads exploits from exploits/exploit-name and keeps track of exploit metadata
        log_manager.py < Writes logs to logs/active/latest.log, handles logserver.pid File
        device_manager.py < Connects to PS5, Sends Data, Recives Responses, Feeds about Panel
    runtime/
        executor.py < Actually runs things, Payload Send, Exploit Execution
        temp/ < Any temp files
        cache/ < Any cached files
exploits
    exploit-name/
        exploit.json < Holds:
            {
            "name": "exploit-name",
            "type": "lua"
            }
        logserver.py < That specific exploits log server
    shared/
        base_exploit.py < Base class for all exploits, defines requires functions
payloads/
    exploit-name/
        *.exploits expected file type, defined by type in exploit.json
    bin/
        raw/ < Raw Shellcode / hex
        elf/ < ELF Binaries
    metadata/
        payload_index.json < Index of all Payloads
        tags.json < Used to fill Exploit Class in the UI
devices/
    ps5/
        connector.py < Opens connection
        protocol.py < Defines how data is sent/received
    base/
        device.py < Base Device Class
        interface.py < Defines requiered devices functions
logs/
    active/
        latest.log < Live log file (UI reads this)
        logserver.pid < PID of the running logfile (kill method between windows and linux differs)
    sessions/
        session-id.log
    errors/
        crash.log
config/
    exploit-name.json < Contains the last ip that a payload was used to, reads from this by default for this exploit, however, on next launch, replace this with default values and ask for the IP/ port again
main.py < Main Executable
.gitignore < Explained in further

Terms:
     Broad:
        Session ID (a randomly generated 6 letter string, ex. a2md2a), but be sure one that was used doesnt get used again
     exploit.json
        exploit-name, the string to be used to identify different exploits
        type, the type of files to expect to be sent (can send others if allow-unsafe-payloads is run)


Further:
    payloads/metadata/manifest.json should be layed out and read like this:
        {
        "payload-name.extension": {
            "exploit": "exploit-name",
            "type": "extension",
            "tags": ["filler", "filler"],

            "exploit": "exploit-name",
